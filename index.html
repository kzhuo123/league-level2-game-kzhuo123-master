Knight Defense

by: Kevin Zhuo

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I was interested in a game where you have to defend against hordes of enemies. I chose this style over other games because I feel that this game has a lot of potential to add new mechanics. Also, I believe that this type of game can be very fun to play as time goes on and the monsters go faster.

2. What I Learned

From the user testing, I learned to incorporate ways to make the game based around a score instead of just kills. This way, people cannot reach an infinite score. One feedback was making the monsters harder to beat, so I added a code that made the monsters spawn faster as time progresses. Another feedback I got was to make the knight move faster so it would be possible to get the harder monsters.

3. Architecture of My Game

Like the League Invaders game, there is a class that runs the game, a class that creates the objects of the game, a class the runs the objects, and a class for the different objects. These classes all need to work for the main game class to work, which creates everything. The smaller individual object classes go under the Object Manager class which then goes under the Game Panel class.

4. Technical Implementation

One part of the game I am proud of is incorporating animations in my game. To get the animations, I had to download a gif from the internet and sort out the individual images. Afterwards, I had to make an array that cycles through all the images. To create the animation whenever I attacked, I had to only make it cycle through the animation when the attack key was pressed.

5. Most Challenging Part

The most challenging part of this project was getting the animations correct. A few difficulties I faced was at first I did not name my animations correctly, so I had trouble sorting them. However, when I redid it, I named each animation according to their order and it was a lot easier to use. Incorporating the animations also required me to use the for loop correctly. At first, I did not know what code I should use to code it but as a named it according to number, I could just use a + 1 at the end to cycle through them.

6. Future Fixes and Updates

Some improvements I would like to make in the future would be to make this game more complicated with power-ups and such. If I was to continue working on this game in the future, I would improve it by adding different enemies and more features in the game to make the average play time last longer.
